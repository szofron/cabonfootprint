# เครื่องคำนวณคาร์บอนฟุตพริ้นท์องค์กร (CFO) — คณะแพทยศาสตร์ มอ.

เว็บแอปไฟล์เดียว (`index.html`) ให้แต่ละหน่วยงานกรอกข้อมูลและคำนวณการปล่อยก๊าซเรือนกระจกตามแนวทาง GHG Protocol / อบก. (TGO) อัตโนมัติ ไม่ต้องติดตั้งโปรแกรม ไม่ต้องมี Excel

## ความสามารถ
- **ล็อกอินรายหน่วยงาน** (เลือกหน่วยงาน + ตั้งรหัสผ่านได้) ข้อมูลแยกกันต่อหน่วยงาน
- กรอกครบ **Scope 1 / 2 / 3 + Carbon Sink** — เลือก EF ด้วยการ **พิมพ์ค้นหา**
- **กรอกข้อมูลแยกรายเดือน** (เลือกเดือน/ปี รวมถึงเดือน-ปีในอนาคตเพื่อวางแผน) หรือกรอกแบบยอดรวมทั้งปีก็ได้ — ข้อมูลเก่าก่อนอัปเดตจะถูกย้ายเป็น "ยอดรวมทั้งปี" อัตโนมัติ
- **Dashboard เลือกช่วงเวลาได้**: รายเดือน / ไตรมาส 1–4 / ครึ่งปีแรก-หลัง / ทั้งปี พร้อมกราฟการปล่อยรายเดือน (Carbon Sink เป็นข้อมูลรายปี — มุมมองบางส่วนคิดตามสัดส่วนเดือน)
- **เปรียบเทียบก่อน-หลังมาตรการลด**: บันทึกข้อมูลก่อนลดเป็น Baseline แล้วระบบเทียบกับข้อมูลปัจจุบันให้อัตโนมัติ ว่าลดลงกี่ kgCO₂e / กี่ % (แยกรายหมวดและรายเดือน)
- คำนวณ kgCO₂e อัตโนมัติ + **Dashboard รายหน่วยงาน** (กราฟโดนัท/แท่ง)
- **Dashboard รวมทุกหน่วยงาน (ภาพรวมคณะ)** — KPI รวม, สัดส่วน Scope, Net รายหน่วยงาน, ตารางเปรียบเทียบ, ส่งออก CSV
- บันทึกอัตโนมัติในเบราว์เซอร์ + ส่งออก/นำเข้า JSON และ CSV
- **(ออปชัน) ซิงค์คลาวด์** ด้วย Firebase → ทุกหน่วยงานส่งข้อมูลเข้าศูนย์กลางอัตโนมัติ ไม่ต้องส่งไฟล์
- **โหมดกลางวัน/กลางคืน (Light/Dark)**: กดปุ่ม 🌙/☀️ มุมขวาบน สลับได้ทุกหน้า ระบบจำค่าไว้ในเบราว์เซอร์และเลือกตามธีมของเครื่องอัตโนมัติในครั้งแรก สี/กราฟ/ตารางปรับให้อ่านชัดทั้งสองโหมด
- ค่า EF อ้างอิง TGO Thai National LCI (CFO ก.พ.2569 / CFP พ.ค.2569)

## ส่วนที่ 1 — นำขึ้น GitHub Pages เพื่อแชร์ (ฟรี)

### วิธี ก) อัปโหลดผ่านเว็บ (ง่ายสุด ไม่ต้องใช้คำสั่ง)
1. เข้าสู่ระบบ GitHub → กด **New repository** ตั้งชื่อเช่น `cfo-medpsu` เลือก **Public** → **Create**
2. กด **Add file → Upload files** ลากไฟล์ `index.html` และ `README.md` เข้าไป → **Commit changes**
3. ไปที่ **Settings → Pages**
4. **Source** เลือก **Deploy from a branch** → **Branch** = `main` โฟลเดอร์ `/(root)` → **Save**
5. รอ ~1 นาที จะได้ลิงก์ `https://<บัญชี>.github.io/cfo-medpsu/` → ส่งให้ทุกหน่วยงานใช้งานได้เลย

### วิธี ข) ใช้ Git (สำหรับผู้ที่ถนัดคำสั่ง)
ในโฟลเดอร์ที่มี `index.html` และ `README.md`:
```bash
git init
git add .
git commit -m "CFO calculator"
git remote add origin https://github.com/<บัญชี>/cfo-medpsu.git
git branch -M main
git push -u origin main
```
แล้วเปิด GitHub Pages ตามข้อ 3–5 ข้างบน

## ส่วนที่ 2 — เปิดใช้ Dashboard รวมศูนย์อัตโนมัติ (Firebase — ฟรี)

ถ้าต้องการให้ทุกหน่วยงาน "ส่งข้อมูลเข้าศูนย์กลางเอง" และผู้ดูแลเห็นภาพรวมคณะแบบเรียลไทม์ (ไม่ต้องส่งไฟล์ JSON):

1. ไปที่ https://console.firebase.google.com → **Add project** ตั้งชื่อ → สร้าง (ปิด Google Analytics ได้)
2. เมนูซ้าย **Build → Firestore Database** → **Create database** → เลือก **Start in test mode** → เลือก region (asia-southeast1) → Enable
3. กดไอคอนเฟือง **Project settings** → เลื่อนลงหา **Your apps** → กดไอคอน **</>** (Web) → ตั้งชื่อ → Register
4. คัดลอกบล็อก `firebaseConfig = { apiKey: "...", ... }`
5. เปิดไฟล์ `index.html` ด้วย Notepad → ค้นหา `const FIREBASE_CONFIG = null;` → แทนที่ `null` ด้วยอ็อบเจกต์ config เช่น
   ```js
   const FIREBASE_CONFIG = { apiKey:"AIza...", authDomain:"xxx.firebaseapp.com", projectId:"xxx", storageBucket:"xxx.appspot.com", messagingSenderId:"123", appId:"1:123:web:abc" };
   ```
6. บันทึกไฟล์แล้วอัปโหลดขึ้น GitHub ใหม่ — เ�
